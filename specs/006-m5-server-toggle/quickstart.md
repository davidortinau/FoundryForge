# Quickstart & Verification: M5 — Local server toggle

Two-layer verification per Constitution II. **Layer A** is dylib-free unit tests (autonomous, every seam). **Layer B** is the real Apple-Silicon DevFlow DOM e2e whose defining proof is an **external out-of-process `curl`** to the bound endpoint. M5 closes with a `Verified:` line; the reviewer is independent of the author (FR-028).

## Prerequisites

- Repo at `net11.0` / `net11.0-macos` (pinned via `global.json` / `Directory.Packages.props`).
- **Layer A**: none beyond the SDK — `tests/FoundryStudio.Tests` is Core-only, dylib-free.
- **Layer B**: an Apple-Silicon Mac; the FL dylib chain bundled/signed (M0 gate green); a model loaded via M3; `curl` available; MAUI DevFlow `0.25.0-dev` (KI-001).

## Layer B — Apple-Silicon DevFlow e2e (the real "wow" + external proof)

Run on the AppKit head with a model loaded. DOM is the sanctioned evidence channel (KI-001).

### B1 — Activate + start + transparent panel (US1, US4)
1. Launch the app; open **Server** nav → route `/server`.
2. Assert via DOM: `server-page` present; `nav-server` **not** disabled; **0** port controls, **0** auth/API-key controls, **0** LAN/remote-bind controls (SC-002/008).
3. Activate `server-toggle` (start). Assert `server-status` transitions `starting → running` (SC-001).
4. Assert `server-endpoint` shows a **real** `host:port` from `Urls` (not a placeholder/hard-coded port); `server-pilot-light[data-state=running]` is lit; `server-routes` lists `/v1/chat/completions` + `/v1/models`; `server-limitations` + `server-scope-note` present; exactly one of `server-request-log` / `server-request-log-omitted` present (SC-001/003/005/006/008/009).
5. Capture the bound port from `server-endpoint` (DOM) for B3.

### B2 — Copy endpoint (US2)
6. Activate `server-endpoint-copy`; assert the clipboard equals `ServerEndpoints.CopyPayload(Urls)` (the exact base URL) and the UI confirms the copy honestly (SC-003).

### B3 — EXTERNAL client proof (US3, SC-004) — the defining check
Run from a **separate process** (genuinely out-of-process), `<port>` from B1/B5:
```bash
# real model response from the loaded model:
curl -sS http://127.0.0.1:<port>/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"<loaded-model-id>","messages":[{"role":"user","content":"Say hello in five words."}]}'

# real model listing:
curl -sS http://127.0.0.1:<port>/v1/models
```
7. Assert the first `curl` returns a **real OpenAI-compatible completion** generated by the loaded model (non-empty `choices`), and `/v1/models` returns a **real** listing (not a stub) (SC-004).

### B4 — Concurrency check (US5, SC-007)
8. With the server running, trigger an in-app model **load/unload** (M3) while toggling the server (and vice-versa). Assert: operations **serialize** through `IModelStateGate` or surface an honest `server-busy` ("busy, try again") — **0** hangs, **0** native crashes, **0** corrupted state. Confirm no `.Result`/`.Wait()` (code review + `NoBlockingInitGuard` green).

### B5 — In-app chat unaffected (US6, SC-005)
9. Exercise in-app chat (M4) with the server **stopped**, then with it **running** — assert chat streams identically in both (chat never depends on the server). Confirm `server-scope-note` states this.

### B6 — Stop really stops (US1/US3, SC-004)
10. Activate `server-toggle` (stop). Assert `server-status` `stopping → stopped`; `server-pilot-light` **not** lit; endpoint/routes no longer presented as live.
11. Re-run the B3 `curl` → assert the connection is **refused/unavailable** (the server is genuinely down) (SC-004).

### B7 — Clean shutdown (FR-018)
12. Quit the app with the server running; relaunch; assert `/server` opens in the honest **stopped** state (no orphaned listener).

### B8 — Request log honesty (US7, SC-009) — PINNED on hardware
13. Per research R2, determine if FL exposes observable request activity. **If yes**: assert the external `curl` from B3 produced a **real** entry in `server-request-log` (not timer-driven). **If no**: assert `server-request-log-omitted` shows the honest note and there are **0** fabricated entries.

### B9 — Accessibility (SC-010, both themes)
14. In **Workshop Daylight** and **Night Forge**: assert every control is labeled + keyboard-reachable; server state is conveyed by text/label (not copper color alone); WCAG AA contrast holds.
